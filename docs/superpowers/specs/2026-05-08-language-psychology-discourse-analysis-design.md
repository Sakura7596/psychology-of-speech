## 1. 项目概述

### 1.1 目标

开发一个基于语言心理学的话语分析多智能体系统，结合 RAG 技术和大语言模型，对自然语言文本进行深度分析，揭示说话者的意识目的、逻辑结构和心理动机。

### 1.2 使用场景

- **当前**：个人学习，掌握语言心理学分析方法和 AI 系统开发
- **未来**：开放为小程序等产品，实现商业化

### 1.3 核心约束

- 中文优先，英文作为补充
- 所有分析严格基于科学理论，避免伪科学
- **系统不提供临床心理诊断，仅做语言特征辅助分析**
- **所有输出必须包含免责声明：非专业心理咨询意见**
- 用户隐私优先，数据处理尽可能在本地完成
- 支持完全离线模式（使用本地 LLM 如 DeepSeek-R1-Distill 或 Qwen2.5）
- 渐进式开发：本地 CLI/Web → 商业化 API

## 2. 系统架构

### 2.1 五层架构

```
┌─────────────────────────────────────────────────────────────┐
│              接口层 (Interface Layer)                         │
│   CLI 命令行 │ FastAPI Web │ 未来: 小程序 API                 │
├─────────────────────────────────────────────────────────────┤
│     Guardrails 层（贯穿所有层）                               │
│   幻觉防御 │ 伦理守则 │ 隐私保护 │ 免责声明注入                │
├─────────────────────────────────────────────────────────────┤
│           多智能体协作层 (Agent Layer)                         │
│  协调器 ──→ 文本解析 │ 心理分析 │ 逻辑推理 │ 报告生成          │
├─────────────────────────────────────────────────────────────┤
│           增强推理层 (Reasoning Layer)                         │
│   LLM API 调用 │ CoT 推理 │ Few-shot 提示 │ 结果校验          │
├─────────────────────────────────────────────────────────────┤
│          RAG 知识引擎层 (Knowledge Layer)                      │
│   向量检索 │ 知识图谱 │ 案例库 │ 理论文献库 │ 元数据过滤        │
├─────────────────────────────────────────────────────────────┤
│        文本处理与特征提取层 (NLP Pipeline)                      │
│   HanLP 句法依存 │ 情感分析 │ 修辞识别 │ 实体提取              │
├─────────────────────────────────────────────────────────────┤
│          基础设施层 (Infrastructure)                           │
│   配置管理 │ 日志/可观测性 │ 缓存 │ 持久化 │ 隐私保护          │
├─────────────────────────────────────────────────────────────┤
│          评测层 (Evaluation)                                   │
│   黄金数据集 │ 回归测试 │ A/B 测试 │ 人类一致性                │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 核心数据流

```
用户输入文本
    ↓
NLP 特征提取（分词、句法、情感、实体）
    ↓
协调器 Agent 分解任务
    ↓
并行调度三个分析 Agent（各自从 RAG 检索知识增强）
    ↓
文本解析 Agent ──┐
心理分析 Agent ──┼──→ 报告生成 Agent → 输出结构化报告
逻辑推理 Agent ──┘
```

### 2.3 插件化设计

所有 Agent 实现统一的 `BaseAgent` 接口：

```python
class BaseAgent:
    def analyze(self, context: AnalysisContext) -> AgentResult
```

新增 Agent 只需：
1. 继承 `BaseAgent`
2. 实现 `analyze` 方法
3. 注册到协调器

### 3.1 关键设计决策

- **LLM 接口抽象**：统一调用接口，切换模型只需改配置，不改代码
- **Embedding 本地化**：使用本地模型，避免每次分析产生 API 费用
- **ChromaDB 嵌入**：零额外部署成本，数据始终在本地
- **NLP 主力 HanLP**：深度学习模型处理复杂句式，jieba 作为快速后备
- **离线模式**：可选本地 LLM（DeepSeek-R1-Distill / Qwen2.5），数据完全不出本地
- **Guardrails 模块**：独立的安全/伦理/隐私防护层，贯穿所有 Agent

## 4. Agent 详细设计

### 4.1 总协调器（Orchestrator）

**职责**：接收用户输入，分解分析任务，调度其他 Agent，合并结果

**工作流程**：
1. 接收用户文本 + 分析偏好（深度、语言）
2. 判断分析类型（简短对话 / 长文本 / 对话历史）
3. **动态路由**：根据文本长度/复杂度自动调整 Agent 参与度和分析深度
   - 短文本（<100字）：跳过深度逻辑分析，仅做基础心理+文本分析
   - 中等文本（100-500字）：全 Agent 参与，标准深度
   - 长文本（>500字）：全 Agent 参与，启用分段分析+交叉验证
4. 生成分析计划（调用哪些 Agent、顺序、参数）
5. 并行调度：文本解析 + 心理分析 + 逻辑推理
6. 收集三个 Agent 结果
7. 传递给报告生成 Agent
8. 返回最终报告

**冲突解决策略**：置信度加权投票
**置信度融合规则**：当多个 Agent 对同一结论给出置信度时，采用加权融合：
- 最终置信度 = Σ(各Agent置信度 × 权重) / Σ权重
- 文本解析权重 0.3，心理分析权重 0.4，逻辑推理权重 0.3
- 任一 Agent 置信度 < 0.3 时，该结论标记为"低可信度"

**输入**：用户文本 + 分析偏好
**输出**：完整分析报告

### 4.2 文本解析 Agent（Text Analyst）

**职责**：语言学层面的结构化分析

**分析维度**：

| 维度 | 具体内容 |
|------|---------|
| 句法结构 | 句子类型（陈述/疑问/祈使/感叹）、从句嵌套、主被动语态 |
| 词汇选择 | 用词正式程度、情感词汇标记、模糊语（大概/可能）识别 |
| 语气与情态 | 情态动词分析（必须/应该/可以）、语气强度 |
| 修辞手法 | 比喻、排比、反问、夸张、讽刺等识别（HanLP 语义角色标注辅助） |
| 话语标记 | "但是""其实""说实话"等转折/强调标记 |

**输入**：原始文本 + NLP 特征提取结果
**输出**：结构化文本分析报告（JSON）

### 4.3 心理分析 Agent（Psychology Analyst）

**职责**：基于语言心理学理论分析说话者心理状态

**理论框架**：

| 理论 | 创始人 | 应用场景 |
|------|--------|---------|
| 言语行为理论 | Austin / Searle | 识别言外行为：断言、指令、承诺、表达、宣告 |
| 会话含义 | Grice | 分析违反合作原则时的隐含意义 |
| 礼貌策略 | Brown & Levinson | 分析面子威胁行为和礼貌补偿策略 |
| 语言相对论 | Sapir-Whorf | 分析语言结构对思维的反映 |
| 依恋理论语言标记 | Bowlby | 通过语言模式推断依恋风格 |
| 认知负荷理论 | Sweller | 通过语言复杂度推断认知状态 |
| **儒家面子理论** | **胡先缙 / 黄光国** | **中国本土面子观：脸（道德面子）vs 面子（社会面子）** |
| **高语境沟通** | **Hall** | **中国文化语境下的隐含信息、非言语依赖、关系导向沟通** |
| **权力距离** | **Hofstede** | **语言中的敬语、称呼策略、权威服从表达** |
| **网络语言心理** | **-** | **弹幕/饭圈/网络梗的心理机制：群体认同、情绪宣泄、身份建构** |

**RAG 检索**：从知识库中检索相关理论和案例，增强分析的理论依据
**输入**：原始文本 + 文本解析结果
**输出**：心理分析报告（含理论引用、置信度评分）

**置信度评分标准**：
- 0.0-0.3：推测性分析，证据不足，仅供参考
- 0.3-0.6：有一定依据，但存在多种解释可能
- 0.6-0.8：较强依据，分析结论较为可靠
- 0.8-1.0：强依据，理论支撑充分，结论可靠

### 4.4 逻辑推理 Agent（Logic Analyst）

**职责**：分析论证结构，识别逻辑问题

**分析维度**：

| 维度 | 具体内容 |
|------|---------|
| 论证结构提取 | 前提 → 推理 → 结论 的完整链路 |
| 逻辑谬误检测 | 稻草人、滑坡、诉诸权威、人身攻击、循环论证等 30+ 种 |
| 隐含前提挖掘 | 说话者未明说但论证依赖的假设 |
| 论证强度评估 | 前提是否支持结论、证据是否充分 |
| 反事实分析 | 如果前提为假，结论是否仍成立 |

**RAG 检索**：检索逻辑谬误案例库，匹配具体谬误模式
**输入**：原始文本 + 文本解析结果
**输出**：逻辑分析报告（含谬误类型、隐含前提列表、论证强度评分）

### 4.5 报告生成 Agent（Report Generator）

**职责**：整合所有分析结果，生成面向用户的结构化报告

**报告结构**：
1. 摘要（一段话总结核心发现）
2. 文本特征分析（来自文本解析 Agent）
3. 心理动机分析（来自心理分析 Agent）
4. 逻辑结构分析（来自逻辑推理 Agent）
5. 综合洞察（交叉分析：语言特征如何反映心理状态）
6. 理论依据与局限性声明
7. 建议与延伸思考

**输出深度**：
- **快速摘要**：1-2 段话总结核心发现，适合快速浏览
- **标准报告**：完整的分节报告，包含各 Agent 分析结果和综合洞察
- **深度分析**：在标准报告基础上增加理论溯源、反事实分析、多角度解读，适合学术研究

**输出格式**（可配置模板）：
- **Markdown**：默认格式，适合终端和文档
- **JSON**：结构化数据，适合程序调用和小程序前端渲染
- **HTML**：富文本格式，适合网页展示
- **自定义模板**：支持 Jinja2 模板引擎，用户可定义输出格式

## 5. RAG 知识库设计

### 5.1 理论文献库

| 分类 | 内容 | 数据来源 |
|------|------|---------|
| 语用学 | 言语行为理论、会话含义、关联理论 | 维基百科、学术摘要、教材节选 |
| 心理语言学 | 语言与认知、情绪与语言、人格与语言 | 学术论文摘要、科普文章 |
| 话语分析 | 批判性话语分析、系统功能语言学 | 开放学术资源 |
| 逻辑学 | 形式逻辑、非形式逻辑、常见谬误 | 逻辑学教材、谬误百科 |
| 社会心理学 | 说服理论、态度改变、群体语言 | 心理学教材、研究摘要 |

**存储方式**：文本分块（500-1000 字/块）→ Embedding 向量化 → ChromaDB 存储
**元数据过滤**：每条文献附带元数据（理论年代、文化区域中/西、置信度/权威性、学科分类），支持检索时按条件过滤

### 5.2 案例库

| 类型 | 说明 |
|------|------|
| 谬误案例 | 每种逻辑谬误配 3-5 个真实对话示例 |
| 心理分析案例 | 典型语言模式与心理状态的对应案例 |
| 修辞案例 | 各种修辞手法的经典用例 |
| 争议对话案例 | 多轮对话中立场变化和策略调整的案例 |

**存储方式**：结构化 JSON + 向量化，支持按类型和标签检索

### 5.3 知识图谱

**核心实体**：理论、概念、语言特征、心理状态、逻辑谬误

**核心关系**：
```
理论 ──应用于──→ 语言特征
语言特征 ──反映──→ 心理状态
心理状态 ──驱动──→ 修辞策略
逻辑谬误 ──属于──→ 谬误类型
理论 ──对比──→ 理论
```

**存储方式**：NetworkX 图结构，序列化为 JSON 文件（规模化后可迁移至 Neo4j，接口兼容设计）

### 5.4 对话案例库（合规数据源）

| 来源 | 用途 | 合规性 |
|------|------|--------|
| 公开数据集（如 LCCC、DuConv 等） | 对话分析基准 | 学术公开数据集 |
| 人工整理的典型对话 | 案例教学和分析示例 | 自行标注 |
| 公开辩论/演讲文本 | 分析说服策略和修辞 | 公开内容，标注来源 |

**数据来源原则**：
- 优先使用合法公开数据集，避免自行爬取带来的法律风险
- 所有数据去除个人信息（用户名、ID 等脱敏）
- 仅保留对话内容和上下文，不存储可识别个人身份的信息

### 5.5 检索策略

```
用户输入文本
    ↓
Embedding 编码（text2vec-base-chinese 或 bge-m3 多语言+密集/稀疏混合检索）
    ↓
┌───────────────────────────────────────┐
│  向量检索（语义相似）                  │
│  + 关键词检索（精确匹配）              │
│  + 图谱检索（关系路径）                │
└───────────────────────────────────────┘
    ↓
结果融合排序 → 作为 Agent 的上下文
```

## 7. 可能遇到的问题与解决方案

| 问题 | 风险等级 | 解决方案 |
|------|---------|---------|
| LLM 幻觉（生成不存在的理论） | 高 | 每条结论必须引用知识库来源；无法找到来源时标注"推测性分析"；guardrails 模块强制验证 |
| 中文 NLP 分词歧义 | 中 | HanLP 深度学习模型 + jieba 自定义词典后备 + LLM 辅助消歧 |
| 心理分析过度解读 | 高 | 强制置信度评分 + 局限性声明；低于阈值的推测不输出；免责声明自动注入 |
| 误用为临床诊断工具 | 高 | guardrails 伦理模块：检测诊断性语言，自动插入免责声明；明确声明"非专业心理咨询" |
| API 调用成本失控 | 中 | 缓存相似查询结果；控制 prompt 长度；设置 token 预算 |
| 知识库内容过时 | 低 | 设计可更新机制，定期补充新文献 |
| 多 Agent 结果冲突 | 中 | 协调器置信度加权融合规则；任一 Agent < 0.3 时标记低可信度 |
| 隐私合规风险 | 高 | 数据本地优先；完全离线模式可选；用户数据不上传第三方；提供数据删除功能 |

## 8. 实施路线图

### 阶段一：基础框架（1-2 周）
- 搭建 Python 项目结构
- 实现 BaseAgent 接口和协调器骨架
- 集成 DeepSeek API
- 配置 Pydantic Settings v2 多环境管理
- **交付物**：可运行的单 Agent demo（输入文本 → LLM 直接分析）

### 阶段二：NLP 管道 + 文本解析 Agent（1-2 周）
- 集成 HanLP（主力）+ jieba（后备）
- 集成 transformers 中文情感/心理模型
- 实现文本解析 Agent 的完整分析维度
- **交付物**：输入文本 → 输出结构化文本分析

### 阶段二附：评测基准构建（0.5-1 周）
- 构建小型评测集（50-100 条带标注对话）
- 覆盖：心理分析、逻辑谬误、修辞识别、情感分析
- 用于后续回归测试和 A/B 测试
- **交付物**：黄金数据集 + 基础评测脚本

### 阶段三：RAG 知识引擎（2-3 周）
- 搭建 ChromaDB 向量库（含元数据过滤）
- 导入基础理论文献和案例
- 实现检索增强推理
- 构建知识图谱（NetworkX）
- **交付物**：分析结果附带理论引用

### 阶段四：心理分析 + 逻辑推理 Agent（2-3 周）
- 实现心理分析 Agent（含中西方理论框架）
- 实现逻辑推理 Agent（含谬误检测）
- 实现 guardrails 模块（幻觉防御、伦理守则、隐私保护）
- **交付物**：三个 Agent 并行分析 + 安全防护

### 阶段五：报告生成 + 协调器完善（1-2 周）
- 实现报告生成 Agent（支持 Markdown/JSON/HTML 多格式）
- 完善协调器动态路由和置信度融合
- 支持多分析深度
- **交付物**：端到端完整系统

### 阶段六：Web 接口 + 可观测性（1-2 周）
- FastAPI Web 服务
- 缓存优化、错误处理
- 集成 LangFuse 或自研日志分析（可观测性）
- Agent 单元测试（mock LLM）+ 集成测试
- **交付物**：可通过 API 调用的完整服务 + 监控面板

**总预估时间**：9-16 周

## 9. 测试策略

| 测试类型 | 目标 | 工具 |
|---------|------|------|
| Agent 单元测试 | 验证每个 Agent 的逻辑正确性（mock LLM 调用） | pytest + mock |
| NLP 管道测试 | 验证分词、句法、情感分析的准确性 | pytest + 黄金数据集 |
| Guardrails 测试 | 验证幻觉防御、伦理守则、隐私保护的有效性 | pytest + 边界用例 |
| 集成测试 | 验证 Agent 间协作和端到端流程 | pytest + FastAPI TestClient |
| 回归测试 | 每次代码变更后用黄金数据集验证分析质量不退化 | pytest + 评测脚本 |
| A/B 测试 | 对比不同 LLM（DeepSeek vs OpenAI）和不同 Agent 策略的效果 | 自研评测框架 |
| 人类一致性测试 | 分析结果与人类专家判断的一致性 | 人工标注 + Cohen's Kappa |

### 10.3 多语言支持说明

当前系统以中文为优先分析对象，但架构上预留了英文/中英混合处理接口：

- 知识库按文化区域（中/西）元数据标记，支持跨语言检索
